## 🎨 VISUAL DESIGN STRATEGY (WEB DESIGNER MODE)

### Design Principle 1: CONSISTENCY ACROSS ECOSYSTEM
- **Color Palette (from Trinity landing page):**
  - Primary: `#667eea` (Purple/Indigo)
  - Secondary: `#764ba2` (Deep Purple)
  - Accent: `#4ade80` (Green - success/blockchain)
  - Danger: `#ef4444` (Red - warnings)
  - Background: `#f5f7fa` (Light) or `#ffffff` (Pure white)

- **Typography:**
  - Headings: System font stack (Segoe UI, Roboto, sans-serif) - professional
  - Body: Same system stack - readable, trustworthy
  - Code: Monospace for technical sections

### Design Principle 2: VISUAL HIERARCHY (WEB DESIGNER MODE)
- **Humanified homepage:** Emphasize VALUES + STORY (not tools)
  - Hero: Large, clear headline about values
  - Services: Secondary hierarchy (learn about tools, not primary focus)

- **Trinity subpage:** Emphasize CAPABILITY + SIMPLICITY
  - Hero: "AI Research You Can Trust"
  - Cards: Feature-focused, clean layout

- **POWER subpage:** Emphasize INTELLIGENCE + COST OPTIMIZATION
  - Hero: "Intelligent Routing for Maximum Margin"
  - Sections: How it works, cost breakdown

### Design Principle 3: MICRO-INTERACTIONS (WEB DESIGNER MODE)
- Buttons: Hover states with gradient shift
- Cards: Subtle shadow on hover, lift animation
- CTAs: Clear affordance (color change on hover)
- Navigation: Smooth transitions, current page indicator

### Design Principle 4: CONVERSION FLOW (WEB DESIGNER MODE)
Every page answers:
1. **CLARITY:** What is this? (Hero section)
2. **RELEVANCE:** Why should I care? (Benefits section)
3. **TRUST:** Can I trust this? (Proof/Testimonials)
4. **ACTION:** What's next? (CTA - multiple entry points)

- **Tablet (640-1024px):** Two columns where appropriate

### Mobile Priorities
- Hero section must work on mobile (clear, readable, tappable)
- Services cards stack vertically
- Navigation: Hamburger menu (mobile) → Horizontal (desktop)
